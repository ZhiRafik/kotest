# KoTest — Online Testing Service

## Описание
KoTest — это сервис для создания и прохождения онлайн-тестов с автоматической проверкой, поддержкой разных типов вопросов и аналитикой результатов.

## Технологии
- **Kotlin** — основной язык разработки
- **Spring Boot 3.3.5** — фреймворк
- **Spring Security** — аутентификация и авторизация (Session-based)
- **Spring Data JPA / Hibernate** — работа с БД
- **PostgreSQL** — база данных
- **Maven** — сборка проекта
- **MapStruct** — маппинг Entity ↔ DTO

## Функциональные возможности

### Роли
- **Преподаватель (TEACHER)** — создание и управление тестами
- **Студент (STUDENT)** — прохождение тестов, просмотр результатов

### Типы вопросов
- Одиночный выбор (Single Choice)
- Множественный выбор (Multiple Choice)
- Текстовый ответ (Text)

### Основной функционал
- Создание тестов с вопросами
- Автоматическая проверка ответов
- Случайная генерация вариантов (перемешивание вопросов и ответов)
- Ограничение по времени прохождения
- Защита от повторной отправки ответов
- Хранение истории попыток и результатов
- Аналитика по тестам (средний балл, распределение оценок)

## Архитектура

### Слои
1. **API** — интерфейсы контроллеров (контракты)
2. **Controller** — обработка HTTP-запросов, валидация
3. **Service** — бизнес-логика
4. **Mapper** — преобразование Entity ↔ DTO
5. **Repository** — работа с БД (Spring Data JPA)

### Модели данных
- `User` — пользователь (студент/преподаватель)
- `Test` — тест (название, описание, длительность, преподаватель)
- `Question` — абстрактный вопрос (текст, баллы, тип)
    - `SingleChoiceQuestion` — одиночный выбор (варианты + правильный ответ)
    - `MultipleChoiceQuestion` — множественный выбор (варианты + набор правильных ответов)
    - `TextQuestion` — текстовый ответ (правильный ответ)
- `Attempt` — попытка прохождения (студент, тест, время начала, завершена/нет)
- `Result` — результат попытки (баллы, процент)

### Стратегия наследования
`SINGLE_TABLE` — все типы вопросов хранятся в одной таблице `question` с дискриминатором `question_type`.

## API Endpoints

### Аутентификация (`/auth`)
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/register` | Регистрация пользователя |
| POST | `/login` | Вход в систему |
| GET | `/me` | Получить текущего пользователя |
| POST | `/logout` | Выход из системы |

### Тесты (преподаватель) (`/tests`)
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/` | Создать тест |
| GET | `/` | Получить все свои тесты |
| GET | `/{testId}` | Получить тест с вопросами |
| PUT | `/{testId}` | Обновить тест |
| DELETE | `/{testId}` | Удалить тест |
| POST | `/{testId}/questions/single` | Добавить вопрос с одиночным выбором |
| POST | `/{testId}/questions/multiple` | Добавить вопрос с множественным выбором |
| POST | `/{testId}/questions/text` | Добавить текстовый вопрос |
| PUT | `/questions/single/{questionId}` | Обновить вопрос (single) |
| PUT | `/questions/multiple/{questionId}` | Обновить вопрос (multiple) |
| PUT | `/questions/text/{questionId}` | Обновить текстовый вопрос |
| DELETE | `/questions/{questionId}` | Удалить вопрос |

### Прохождение тестов (студент) (`/tests`)
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/available` | Список доступных тестов |
| POST | `/{testId}/start` | Начать тест (создать попытку) |
| POST | `/attempts/{attemptId}/submit` | Отправить ответы |

### Результаты (`/results`)
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/my` | Мои результаты (студент) |
| GET | `/my/{testId}` | Мои результаты по тесту (студент) |
| GET | `/test/{testId}` | Результаты всех студентов (преподаватель) |
| GET | `/test/{testId}/analytics` | Аналитика по тесту (преподаватель) |

## Настройка и запуск

### Требования
- Java 21+
- PostgreSQL 14+
- Maven 3.8+

### Настройка БД
```sql
CREATE DATABASE kotest;
```
### Тесты Postman лежат в
```
KoTest API.postman_collection.json
```
